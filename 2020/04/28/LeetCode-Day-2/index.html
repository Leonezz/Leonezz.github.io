进阶:

  • 使用 O(n) 空间复杂度的解法很容易实现。
  • 你能想出一个只使用常数空间的解决方案吗？

y1s1标着$Hard$的题目就是$Hard$，一开始想着一步到位，直接交换两个最合适的节点，一次修正二叉查找树。后来想了半天也没做出来，看了看98题的题解发现找到错误的两个节点还是可以做到的，但是直接交换两个错误的节点得到的可能仍是错误的二叉查找树。基于一个朴素的猜想：经过有限次的查找替换一定能把BSD纠正过来，利用98题的题解做出了这题。
主要还是上题用到的中序遍历，只不过在找到错误的时候，先把错误的两个节点交换，再返回false，这样经过有限次迭代，相信可以纠正二叉查找树。不过中序遍历的空间复杂度是$O(log n)$的，没有做到题目说的常数空间复杂度。

[101]对称二叉树

Category Difficulty
$algorithms$ $Easy (50.83\%)$
Tags tree | depth-first-search | breadth-first-search

给定一个二叉树，检查它是否是镜像对称的。

例如，二叉树 $[1,2,2,3,4,4,3]$ 是对称的。


但是下面这个 $[1,2,2,null,3,null,3]$ 则不是镜像对称的:

进阶：

你可以运用递归和迭代两种方法解决这个问题吗？

思路

标着$Easy$的题目，但我感觉还挺难的。刚开始写了个判断父子级二叉节点是否对称的函数，发现拓展一些就可以作为递归解法解出来，然后根据尾递归的过程推出了迭代的解法。

递归过程主要就是判断两个节点p,q值是否相等，若相等判断p->left和q->right以及p->right和q->left是否相等。

根据递归过程，可以使用一个堆栈，每次把p->left和q->right以及p->right和q->left按顺序压入堆栈，比较时一次弹出两个节点比较，这样也实现了左右交叉比较。

[102]二叉树的层序遍历

Category Difficulty
$algorithms$ $Medium (61.64\%)$
Tags tree | breadth-first-search

给你一个二叉树，请你返回其按 层序遍历 得到的节点值。 （即逐层地，从左到右访问所有节点）。

示例：
二叉树：$[3,9,20,null,null,15,7]$,

返回其层次遍历结果：

思路

这题虽然没单纯的层序遍历那么简单，但也每增加多少难度。主要就是如何在层序遍历的同时分清楚各层的元素。自然地想到用一个数字curLevelNodesCnt来标记当前层的剩余节点数，当前层节点数剩余为0时，就新建一个向量作为下一层的容器。同时层序遍历在访问当前层的时候就已经把下层的节点放入队列了，所以还需要一个数字nextLevelNodesCnt来记录下一层的节点个数。这样当当前层的剩余个数curLevelNodesCnt为0时，下层个数也统计好了，只需要把nextLevelNodesCnt赋值给curLevelNodesCnt，然后把nextLevelNodesCnt清零即可。
